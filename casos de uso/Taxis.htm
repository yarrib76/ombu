Caso de Uso: CU-01 Dando de Alta Vehículos

Actor: Clientes

Curso Normal

Alternativas

1. El usuario ingresa al link Alta de Vehículos

 

2. Se valida según token el tipo de servicio

2.1 Si el usuario no tiene un servicio que permita agregar vehículos se informa.

3. Se solicitan datos del Vehículo. (Especificar tipo. Taxi, Remis, Camion, etc)

3.1 En el campo modelo se puede buscar modeles existentes o crear propios

4. Se confirma el alta

4.1 Si hay datos mal ingresados, el formulario indica los errores encontrados.

5. Se crea vehículos

 

 

Caso de Uso: CU-02 Modificando Datos de Vehículos

Actor: Clientes y Operadores

Curso Normal

Alternativas

2. Se valida según token el tipo de servicio

2.1 Si el usuario no tiene un servicio que permita modificar vehículos se informa.

3. Seleccionar Auto a Modificar

 

4. Cambiar los campos necesarios

4.1 Si hay datos mal ingresados, el formulario indica los errores encontrados.

5. Se modifica datos del vehículos

Caso de Uso: CU-08 Compro Servicios

Actor: Clientes

Curso Normal

Alternativas

1. El usuario ingresa al link Compra de Servicios

 

2. Se muestra en el formulario token del cliente

 

3. Selecciona 1 o más servicios

 

4. Se solicita medio de pago

4.1 Se valida medio de pago. En caso de ser incorrecto se informa.

5. Se confirma la compra

 

 

Caso de Uso: CU-09 Dando de Alta Alertas

Actor: Clientes

Curso Normal

Alternativas

1. El usuario ingresa al link Alta de Alertas

 

2. Se valida según token el tipo de servicio

2.1 Si el usuario no tiene un servicio que permita agregar Alertas se informa.

3. Se solicitan datos de Alerta:

·          Nombre

·          Tipo (Taller o Control de Vencimientos)

·          Fecha y Hora de Aviso (Depende el Tipo)

·          Kilometraje (Depende el Tipo)

3.1 Se verifica que no exista otra alerta con el mismo nombre.

4. Se confirma la alta

4.1 Si hay datos faltantes o  mal ingresados, el formulario indica los errores encontrados.

5. Se crea Alerta

Caso de Uso: CU-12 Búsqueda

Actor: Clientes y Operadores

Curso Normal

Alternativas

1. El usuario selecciona el Botón Busqueda

 

2. Seleccionar tipo de búsqueda:

·          Chofer

·          Vehículos

·          Alertas

·          Clientes

2.1 Según el tipo, se pueden buscar Choferes, Vehículos, Alertas, Clientes.

3. Confirmar Búsqueda

3.1 Si la búsqueda es exitosa se muestra la información  encontrada. En caso contrario muestra mensaje Datos no encontrados

 

Caso de Uso: CU-13 Procesa Alerta Por Vencimientos

Actor: Clientes y Sistema

Curso Normal

Alternativas

1. Se verifica tabla de Vencimientos

·          Estado = Habilitado, No Habilitado

·          Tipo ( Lic Taxi, Tanques Gas)

·          Fecha de Vencimiento

1.1     En caso de no estar habilitado no se procesa el Vencimiento

1.2     Si está habilitado y la fecha del día es igual a la Fecha de Vencimiento se dispara la Alerta

2. Envía mail al destinatario del aviso

 

3. Actualiza tabla con nueva fecha de vencimiento

 

 

Caso de Uso: CU-14 Procesa Alerta Por Mantenimiento

Actor: Clientes y Sistema

Curso Normal

Alternativas

1. Se verifica tabla de Mantenimiento

·          Estado = Habilitado, No Habilitado

·          Tipo (define el tipo de mantenimiento, Cambio de Aceite, Correas, etc.)

·          Periodo (Kilómetros)

1.1     En caso de no estar habilitado no se procesa el Mantenimiento

1.2     Si está habilitado y el kilometraje es igual o mayor a periodo se dispara Alerta.

2. Envía mail al destinatario del aviso
